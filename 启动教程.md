给你一份“队友新电脑从 0 启动”流程，照着做就能起来。

1. 先装环境（Windows）
1. Docker Desktop（必须）
1. Java 17 或 21
1. Maven 3.9.x
1. Git（可选，但建议）
1. Node 不强制（因为前端走 Docker），但装了也行

1. 拉代码并进入项目根目录
1. git clone 你们仓库
1. 进入 health-app 目录

1. 第一次启动（关键两步）
1. 先在本机打后端 jar（因为后端镜像现在直接复制 jar）
   mvn -DskipTests package -f backend/pom.xml

1. 再启动所有容器
   docker compose up -d --build

1. 查看状态
   docker compose ps

1. 验证是否成功
1. 后端健康检查
   curl http://localhost:18080/api/health
   看到 status ok 就对了

1. 前端页面
   浏览器打开 http://localhost:5173

1. 页面使用
1. 先注册并登录（例如 demo / demo123）
1. 登录后新增运动记录
1. 刷新页面记录还在，说明 PostgreSQL 持久化正常

1. 队友最常见坑
1. 在错误目录执行命令
   必须在 health-app 根目录执行 docker compose

1. 命令前误加符号
   不要输入 • 这种项目符号

1. Docker 拉镜像失败
   先执行 docker login，再重试 docker compose up -d --build

1. 端口被占用
   改环境变量后重启：
   BACKEND_PORT、WEB_PORT、POSTGRES_PORT、AI_SERVICE_PORT

1. 一键重置重来（出问题时）
   docker compose down -v
   mvn -DskipTests package -f backend/pom.xml
   docker compose up -d --build

三、不同端改代码后，最小重启命令

1. 只改前端

- 本地模式：web 下 npm run dev 自动生效
- 容器模式：docker compose up -d --build web

1. 只改后端 Java 代码

- 先打包 jar：mvn -DskipTests package -f backend/pom.xml
- 再重建后端容器：docker compose up -d --build backend

1. 只改 AI 服务

- 容器重建：docker compose up -d --build ai-service

1. 改了数据库相关配置

- 一般先重启 backend + postgres
  docker compose up -d --build backend postgres
- 如果要清空数据库重来（会丢数据）
  docker compose down -v
  然后重新 up

四、提交前检查清单（建议复制到群公告）

1. docker compose ps 全部 Up
2. 后端健康检查 200
3. 前端能注册、登录、增删运动记录
4. 后端改动已执行 mvn -DskipTests package
5. 没有把本地临时配置误提交
